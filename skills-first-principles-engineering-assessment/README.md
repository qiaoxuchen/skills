# 第一性原理工程评估 (First-Principles Engineering Assessment)

这是一个专用于防范大模型“瞎重构”、“过度设计”与“噪音修改”的 **硬核护栏 (Guardrail) 技能**。

## 为什么需要这个技能？ (The Pain Points)

市面上大多数通用 AI 助手在处理工程时存在致命缺陷：
1. **炫技型重构**：稍微问点关于代码的问题，就开始搞各种华而不实的抽象。
2. **Git Diff 爆炸**：习惯性地对未牵涉逻辑区域进行格式美化（括号折叠、空行增减），导致提交记录难以审查。
3. **不顾边界**：不分析环境约束、无视原项目奇葩但刚需的硬伤历史逻辑。
4. **废话过多**：无论什么问题都要重述三遍，大量输出无用 Token。

本文件旨在把大模型的作用局限在一个**严苛的工程师职级框架**内。它强行规定了 AI 解决问题时的行为：以事实为准绳，给出成本和可扩展的对比方案，并在执行时死守「最小改动」的红线。

## 技能特性

* **按需触发，不干扰日常聊天**：仅当你的 Prompt 里出现 `评审`、`review`、`重构方案`、`最小可行实施计划`、`验证证据` 等特定关键词时，本技能才会苏醒发威。
* **强制分离要素**：要求 AI 强制剥离输入要求中的：事实 (Facts)、假设 (Assumptions)、限制条件 (Constraints)。
* **多方案博弈**：遇到非平凡问题，必须强制输出至少两个对照方案（最小可行 A 与 高扩展 B），并且默认**强制应用 A 方案**。
* **干净的输出格式**：禁绝大模型输出完整的、动辄几百行的代码；使用三段式的标准评估模板进行回复。

## 如何使用

您可以将本目录中的 `SKILL.md` 直接嵌入到支持外部 Skill 调用的 Agent 框架中（例如放置在对应系统约定的 `.agents/skills/` 目录下）。

当需要 AI 帮你检视代码或者修复严重架构级 Bug 时，请在指令中带上如：
> “帮我 **review** 一下这个并发服务的内存泄漏隐患。”
> “用已有代码出个 **最小可行实施计划** 修复问题。”

此时，该第一性原理技能会立刻接管后续的工作流。
