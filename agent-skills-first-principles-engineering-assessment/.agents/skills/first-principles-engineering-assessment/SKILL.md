# 第一性原理工程评估技能

## 1. 目标
1.1 以事实和约束为基础完成分析与实现。  
1.2 输出可验证、可复现、可审查。  
1.3 在正确性、安全性与可维护性之间做可解释取舍。

## 2. 配套资源
2.1 验证清单：`references/checklists.md`  
2.1.1 用于交付前核对正确性、回归、安全、性能、回滚项。  
2.1.2 使用时机：代码改动、架构调整、排障、发布评审。  

2.2 报告脚本：`scripts/generate_delivery_report.py`  
2.2.1 用于生成标准化交付报告骨架。  
2.2.2 使用时机：进入交付阶段时。  
2.2.3 示例：
```bash
./scripts/generate_delivery_report.py "修复角度检测误报" -o delivery_report.md --risk P1
```

## 3. 适用范围
3.1 适用场景  
3.1.1 需求模糊、存在多解的技术任务。  
3.1.2 需要在正确性、成本、性能、维护性间做取舍。  
3.1.3 改代码、设计方案、排障、review。  

3.2 非适用场景  
3.2.1 纯创意文案或无工程约束讨论。  
3.2.2 仅信息查询且不涉及实现决策。

## 4. 输入契约
4.1 接手任务先确认：  
4.1.1 成功标准（目标结果）。  
4.1.2 约束条件（时间/性能/兼容/资源/合规）。  
4.1.3 当前状态（已有实现/已知问题/可用数据）。  
4.1.4 变更范围（可改与不可改区域）。  

4.2 缺失关键信息时，先提问再执行。

## 5. 标准执行流
5.1 需求澄清  
5.1.1 重述为 `输入 -> 处理 -> 输出`。  
5.1.2 写明非目标（不做什么）。  
5.1.3 通过条件：关键术语有定义，成功标准可观测。  

5.2 第一性拆解  
5.2.1 拆分为 `facts / assumptions / constraints`。  
5.2.2 对假设给出验证方式和置信度。  
5.2.3 拦截条件：核心结论仅由假设支撑且无验证路径。  

5.3 方案收敛  
5.3.1 至少给两个方案：A 最小可行，B 高扩展。  
5.3.2 对比维度：开发成本、维护成本、性能、失败模式、回滚难度。  
5.3.3 决策规则：默认选最小可行；选复杂方案需给不可替代理由。  
5.3.4 冲突优先级：正确性与安全性 > 可维护性 > 性能 > 优雅性。  

5.4 最小化实施  
5.4.1 输出改动清单：文件/函数、行为变化、不改动项。  
5.4.2 执行约束：最小影响面、禁止无关重构、保持原风格。  

5.5 防御式实现  
5.5.1 输入校验与类型约束。  
5.5.2 超时、重试、异常处理。  
5.5.3 资源释放与并发边界控制。  
5.5.4 可观测性：错误码、日志上下文、关键指标。  
5.5.5 触发条件：涉及 IO/网络/并发/数据库/外部 API 时必选。  

5.6 验证与交付  
5.6.1 至少完成一种验证：自动化测试/手工关键路径/风险回归清单。  
5.6.2 建议：优先用 `references/checklists.md` 留痕。  
5.6.3 交付前建议用脚本生成报告骨架。  
5.6.4 交付必须包含：改动摘要、验证证据、残余风险。  

## 6. 风险分级
6.1 P0（阻断级）：数据错误、核心路径不可用、安全风险。  
6.1.1 动作：立即阻断上线或回滚。  

6.2 P1（高风险）：功能可用但结果不稳定、回归概率高。  
6.2.1 动作：限制发布范围，补齐验证后放量。  

6.3 P2（一般风险）：局部体验或非关键性能问题。  
6.3.1 动作：纳入迭代计划并明确修复窗口。  

## 7. 反盲从规范
7.1 发现硬伤时必须给出：问题、影响、证据、替代方案。  
7.2 禁止仅凭经验否定方案，必须可验证。  
7.3 存在争议时，提交最小可验证实验（MVE）裁决。  

## 8. 禁止行为
8.1 仅因“更优雅”进行大规模重构。  
8.2 需求不完整直接编码。  
8.3 未验证宣称“已修复”或“无风险”。  
8.4 明知假设薄弱仍迎合执行且不提示风险。  

## 9. 输出模板
```md
1. 问题定义
1.1 目标：
1.2 非目标：
1.3 约束：

2. 事实与假设
2.1 事实：
2.2 假设（含验证方式）：

3. 方案对比与决策
3.1 方案 A（最小可行）：
3.2 方案 B（高扩展）：
3.3 选择与理由：

4. 改动计划
4.1 文件/函数：
4.2 行为变化：
4.3 不改动项：

5. 验证结果
5.1 测试/验证步骤：
5.2 结果：
5.3 未验证项：

6. 风险分级与处置
6.1 等级（P0/P1/P2）：
6.2 处置动作：
6.3 回滚条件：
```
