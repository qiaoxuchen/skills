# AGENTS.md（Short）

## 1. 作用范围
适用于本工作区 `CURRENT-WORKSPACE` 下所有子目录的技术任务（分析、编码、重构、调试、评审）。

## 2. 规则优先级
1. 系统/平台规则。
2. 用户当次明确指令。
3. 本文件（AGENTS.md）。
4. Skill 细则（`.agents/skills/.../SKILL.md`）。

## 3. Skill 调用策略（降 token）
1. 默认不调用 Skill，优先直接执行最小可行流程。  
2. 仅在出现以下关键词时调用默认 Skill：`评审`、`review`、`重构方案`、`最小可行实施计划`、`风险分级`、`验证证据`、`回滚方案`。  
3. 简单问答、术语解释、纯信息查询、纯文案改写、无工程决策轻量任务，不调用 Skill。  
4. 读取范围遵循“最小必要”：先读目标文件，证据不足再扩读。  

## 4. 输出策略（降 token）
1. 默认输出“结论 + 关键过程日志 + 必要证据”。  
2. 非必要不展开完整推理链，不重复粘贴长文本。  
3. 结构化输出统一使用三级编号：`1.` / `1.1` / `1.1.1`，同层连续编号。  

## 5. 硬性规则
1. 不得盲目迎合技术上站不住脚的要求。  
2. 无验证证据不得宣称“已修复”。  
3. 不得做与目标无关的重构或格式化。  
4. 假设必须显式说明，并给出验证路径。  
5. 非平凡问题至少提供 A/B 两案并给取舍理由。  

## 6. 最小执行流
1. 澄清：重述为 `input -> processing -> output`，明确目标/非目标/约束。  
2. 拆解：区分 `facts / assumptions / constraints`。  
3. 实施：保持最小改动（minimal diff），遵循原风格。  
4. 验证：优先测试；无法测试时给可复现验证步骤。  
5. 交付：必须包含改动摘要、验证证据、残余风险。  

## 7. 评审附加要求
当用户说“检查一下/评审/review”时：  
1. 重新读取当前文件，不依赖历史记忆。  
2. 先给问题清单（按严重级），再给总结。  
3. 明确未验证项与风险边界。
